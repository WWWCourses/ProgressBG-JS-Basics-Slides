Introduction to JS Development Tools

Introduction to JS Development Tools

Introduction to JS Development Tools

JavaScript tooling enhances development workflow by automating tasks, optimizing performance, and improving code quality.
These tools include transpilers, bundlers, linters, and package managers, which streamline various aspects of development.

Linters and Formatters

Linters and Formatters

Overview

Linter like ESLint, is a tool that analyzes JavaScript/TypeScript code for potential errors and enforces coding standards.:
ESLint catches syntax errors, unused variables, and potential bugs before running the code.
It enforces a consistent coding style across the project, like ensuring all functions use camelCase for naming.
Can autofix some issues but is mainly for code quality and consistency.
Formatter like Prettier is a tool that ensures consistent code style (e.g., indentation, semicolons, quotes)
It automatically reformats code but does not catch logical errors or bad practices.

Setup ESLint in VSCode

Install ESLint VSCode Extension (Search for ESLint and install it.)
Install ESLint in Your Project (or as global package)

                npm install eslint --save-dev
Initialize ESLint and create config file

                npx eslint --init
You can modify or extend this configuration by adding your own rules.

                export default [
                    {
                        languageOptions: { globals: globals.browser },
                    },
                    pluginJs.configs.recommended,
                    {
                        rules: {
                            // Enforces the use of double quotes for strings and throws an error if violated
                            "quotes": ["error", "double"],
                        },
                    },
                ];
Enable Auto-fixing on Save (Modify VSCode settings.json)

                {
                    "editor.codeActionsOnSave": {
                        "source.fixAll.eslint": true
                    }
                }

Use ESLint

Run ESLint Manually

                npx eslint .
Fix ESLint Issues Automatically

                npx eslint . --fix

Setup Prettier in VSCode

Install Prettier VSCode Extension (Search for Prettier and install it.)
Install Prettier in Your Project

                npm install --save-dev prettier
Optional: Create a Prettier Configuration File: .prettierrc

                {
                    "semi": true, // Add semicolons at the end of statements
                    "singleQuote": true, // Use single quotes instead of double quotes
                    "trailingComma": "es5" // Add trailing commas where valid in ES5 (e.g., objects, arrays)
                }
Enable Auto-formatting on Save (Modify VSCode settings.json)

                {
                    "editor.formatOnSave": true // Automatically format the code when saving a file
                }

Use Prettier

Run Prettier Manually

                npx prettier --write .
Run Prettier on Specific Files

                npx prettier --write "src/**/*.js"

Optional: Integrate ESLint with Prettier

Install ESLint config for Prettier

                npm install --save-dev eslint-config-prettier
Modify ESLint Config for Prettier

                export default [
                    {
                        extends: [
                            "eslint:recommended", // Includes the recommended ESLint rules for JavaScript
                            "prettier",           // Disables ESLint rules that conflict with Prettier's formatting
                        ],
                    },
                ];

Transpilers

Transpilers

Overview

Transpilers convert modern JavaScript (ES6+) code into a version that is compatible with older browsers.
Common transpilers include Babel, which allows you to use the latest JavaScript features without worrying about browser support.

                // Example of Babel transpiling ES6 code to ES5
                // Original ES6 code
                const greet = () => console.log('Hello, World!');

                // Transpiled ES5 code by Babel
                var greet = function() {
                    console.log('Hello, World!');
                };
You can play live in https://babeljs.io/repl

Bundlers

Bundlers

Overview

Bundlers like Webpack and Parcel combine multiple JavaScript files into a single file or a few files.
This process reduces the number of HTTP requests needed to load your application, improving performance.
Webpack is a highly configurable bundler, allowing you to manage various assets (JavaScript, CSS, images) and optimize them for production.
Parcel, on the other hand, is a zero-config bundler, making it easy to get started without needing extensive setup.

                // Example of a simple Webpack configuration
                const path = require('path');

                module.exports = {
                    entry: './src/index.js',  // Entry point for the application
                    output: {
                        filename: 'bundle.js',  // Name of the bundled file
                        path: path.resolve(__dirname, 'dist')  // Output folder
                    },
                    module: {
                        rules: [
                            {
                                test: /\.css$/,  // Use this rule for CSS files
                                use: ['style-loader', 'css-loader']  // Apply style-loader and css-loader to bundle CSS
                            },
                            {
                                test: /\.(png|jpg|gif)$/,  // For images
                                use: ['file-loader']  // Use file-loader to handle image files
                            }
                        ]
                    },
                    mode: 'production'  // Set Webpack to production mode to optimize the build
                };

Using Webpack to Bundle JavaScript and Assets

Install Dependencies and create Project Structure

You need Webpack and some other dependencies

                npm init -y
                npm install --save-dev webpack webpack-cli style-loader css-loader file-loader
Create next project structure:

                my-webpack-project/
                ├── dist/
                ├── src/
                │   ├── index.js
                │   ├── styles.css
                │   └── logo.png
                ├── index.html
                ├── package.json
                └── webpack.config.js

Create webpack.config.js

Create a webpack.config.js in the root of your project folde

                const path = require('path');

                module.exports = {
                    entry: './src/index.js',  // Entry point for the JavaScript file
                    output: {
                        filename: 'bundle.js',  // Output file name
                        path: path.resolve(__dirname, 'dist')  // Output folder
                    },
                    module: {
                        rules: [
                        {
                            test: /\.css$/,  // Handle CSS files
                            use: ['style-loader', 'css-loader']
                        },
                        {
                            test: /\.(png|jpg|gif)$/i,  // Handle image files
                            type: 'asset/resource',
                        }
                        ]
                    },
                    mode: 'development'  // You can change to 'production' for production builds
                };

Create src/index.js

This is your main JavaScript file, where you'll import the CSS and images.

                import './styles.css';  // Importing the CSS file
                import logo from './logo.png';  // Importing the image

                const imgElement = document.createElement('img');
                imgElement.src = logo;
                document.body.appendChild(imgElement);

                const h1 = document.createElement('h1');
                h1.textContent = 'Welcome to Webpack!';
                document.body.appendChild(h1);

Create src/styles.css

This is a simple CSS file.

                body {
                  font-family: Arial, sans-serif;
                }

                h1 {
                  color: #4CAF50;
                }

                img {
                  width: 200px;
                  height: auto;
                }

Create index.html

This is the basic HTML file that will reference the output JavaScript file (bundle.js).

                <!DOCTYPE html>
                <html lang="en">
                <head>
                    <meta charset="UTF-8">
                    <meta name="viewport" content="width=device-width, initial-scale=1.0">
                    <title>Webpack Example</title>
                </head>
                <body>
                    <script src="dist/bundle.js"></script>  <!-- Link to the bundled JS -->
                </body>
                </html>

Add scripts to package.json (if you have not)

Make sure to add a build script to your package.json to run Webpack.

                {
                  "name": "my-webpack-project",
                  "version": "1.0.0",
                  "main": "index.js",
                  "scripts": {
                    "build": "webpack"
                  },
                  "devDependencies": {
                    "webpack": "^5.0.0",
                    "webpack-cli": "^4.0.0",
                    "css-loader": "^6.0.0",
                    "style-loader": "^3.0.0",
                    "file-loader": "^6.0.0"
                  }
                }

Build the Project

Now you can build the project using:

                npm run build
This will bundle your JavaScript, CSS, and image files into the dist folder, and Webpack will output the bundle.js file.

Package Managers

Package Managers

Package Managers

Package managers like npm and Yarn help you manage dependencies in your JavaScript projects.
They allow you to easily install, update, and remove libraries, keeping your project organized and up-to-date.

                # Installing a package with npm
                npm install lodash --save
